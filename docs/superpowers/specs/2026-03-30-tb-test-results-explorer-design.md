# TB Test Results Explorer — Design Spec

## Overview

An interactive explorer for Bjørn Nyland's EV test data. Not a branded product — a utility that makes his publicly available spreadsheet data easier to browse, compare, and understand.

**Target users:** EV enthusiasts (especially Bjørn's audience) and prospective EV buyers researching real-world performance data.

**Identity:** "TB Test Results Explorer" — uses Bjørn's own "TB" naming convention. Clear "Data by Bjørn Nyland" attribution. Clean, editorial aesthetic. The data is the star, not the wrapper.

## Architecture

### Tech Stack

- Next.js 16 (App Router) with React 19
- TypeScript
- Tailwind CSS 4 + shadcn/ui for interactive form controls
- Recharts for data visualization
- jsPDF for PDF export

### Routing

```
/                               → Dashboard (overview stats + highlights)
/tests/[slug]                   → Individual test sheet (range, noise, acceleration, etc.)
/vehicles/[slug]                → Vehicle profile (all test results for one car)
/compare                        → Weighted ranker (priority sliders → ranked list)
/compare/[slugA]/[slugB]        → Head-to-head comparison (2 vehicles, side-by-side)
```

### Key Architectural Changes from Current Codebase

1. **Monolithic page.tsx → 5 route pages** — each page has focused responsibility, URL = state
2. **Components grouped by feature** — not flat in a single `components/` directory
3. **Data moves from hardcoded TS to generated JSON** — produced by an import script, consumed through a typed access layer
4. **Typed data access layer (`lib/data/`)** — replaces direct array manipulation scattered across components
5. **All navigation becomes URL-based** — no more `activeSheet` state managing which view is shown

## Data Layer

### Static JSON Files

Generated by the import script, one file per test sheet:

```
data/
├── tests/
│   ├── range.json
│   ├── acceleration.json
│   ├── noise.json
│   ├── braking.json
│   ├── weight.json
│   ├── banana.json
│   ├── sunday.json
│   ├── 1000km.json
│   ├── 500km.json
│   ├── arctic-circle.json
│   ├── bangkok.json
│   ├── degradation.json
│   ├── geilo.json
│   ├── zero-mile.json
│   └── meta.json              → Sheet metadata (name, slug, unit, bestDirection, mainColumn)
└── vehicles.json               → Unified vehicle list with slugs and test participation
```

### Import Script (`scripts/import-sheets.ts`)

- Fetches each sheet tab as CSV via the public export URL (`https://docs.google.com/spreadsheets/d/{ID}/export?format=csv&gid={GID}`) — no API key needed
- Parses CSV into structured JSON
- Generates URL-safe slugs for each vehicle (e.g., `tesla-model-y-lr-awd`)
- Cross-references all test sheets to build `vehicles.json` — a unified vehicle list with which tests each vehicle appears in
- Run manually: `npm run import-data`

### GitHub Action (`.github/workflows/update-data.yml`)

- Runs on a weekly cron schedule
- Executes `npm run import-data`
- If JSON files changed, commits and pushes the updated data
- Can also trigger a deployment rebuild (Vercel/Netlify webhook)
- No secrets required — uses the public CSV export URL, no Google API key needed

### Data Access Layer (`lib/data/`)

All components consume data through these functions — no direct JSON imports in components:

- `getTests()` → list of all test sheets with metadata
- `getTestBySlug(slug: string)` → single test sheet with all rows, typed
- `getVehicles()` → all vehicles with slugs and basic info
- `getVehicleBySlug(slug: string)` → one vehicle with all its test results across every sheet, plus rank/percentile per test
- `getVehiclesForComparison(slugA: string, slugB: string)` → two vehicles with their shared test data, formatted for side-by-side display

### Scoring Utilities (`lib/utils/scoring.ts`)

- `normalizeScore(value, min, max, direction)` → 0-100 score for radar charts and ranking
- `computeWeightedRank(vehicles, weights)` → sorted list for the weighted ranker
- Extracted from components where this logic currently lives inline

## Features

### Existing (Preserved + Cleaned Up)

All current features are retained during the refactor:

- **Dashboard** — overview stats across all test categories, car comparison radar chart
- **Test sheet views** — stats bento, top 10 chart, full sortable/filterable data table
- **Sheet-specific visualizations** — acceleration scatter, noise histogram, range efficiency scatter, degradation scatter, WLTP vs reality, winter penalty chart
- **Search** — real-time text filtering across all columns
- **Sorting** — click-to-sort with numeric vs text awareness
- **Export** — CSV and PDF export
- **Theme** — light/dark mode with system preference detection

### New: Weighted Ranker (`/compare`)

A "which EV is right for me?" tool using simple priority sliders.

- **4-6 metric sliders:** Range, Cargo (banana boxes), Noise, Acceleration, Braking, Weight
- Each slider ranges 0-100, representing how much the user cares about that metric
- **Ranked results list** below the sliders, sorted by weighted score
- Each vehicle row shows: name, individual metric scores, overall weighted score
- Updates in real-time as sliders move
- Clicking a vehicle row navigates to its profile page (`/vehicles/[slug]`)
- Only vehicles that have data for all selected metrics appear in the ranking

### New: Head-to-Head Comparison (`/compare/[slugA]/[slugB]`)

Side-by-side comparison of exactly 2 vehicles.

- **Vehicle pickers** at the top — two autocomplete inputs, pre-filled from URL slugs
- **Side-by-side metric rows** for every test both vehicles appear in:
  - Layout: `value | bar visualization | value`
  - Better value highlighted per row
- **Winner indicator** per metric
- **Shareable URL** — updates as vehicles are changed via the pickers
- Navigating here from a vehicle profile pre-fills one of the two slots

### New: Vehicle Profile (`/vehicles/[slug]`)

All test results for a single vehicle in one place.

- **Header:** vehicle name, type badge (car/van)
- **Summary card:** key stats at a glance — range, 0-100 time, noise, weight, cargo
- **Full results table:** every test this vehicle appears in, showing:
  - Test name
  - Vehicle's value
  - Rank position (e.g., "12th of 87 tested")
  - Percentile
- **"Compare with..." button** — navigates to head-to-head with this vehicle pre-selected

## Visual Design

### Identity

- **Name:** TB Test Results Explorer
- **Byline:** "Data by Bjørn Nyland"
- **Tone:** Clean, editorial, data-first. Not a branded product — a utility.

### Design Direction

- Refine the current aesthetic rather than redesign from scratch
- Warm stone/neutral palette (light mode), clean dark slate (dark mode)
- Consistent spacing, typography, and component patterns via shared UI primitives
- shadcn/ui for interactive controls (sliders, autocomplete, select) — consistent, accessible, minimal custom CSS
- Existing chart visualizations retained with consistent color palette

### Typography

- Keep Inter (body) and Space Grotesk (labels/small caps) — they work well together
- Ensure consistent type scale across all pages

## Out of Scope

- User accounts or saved preferences
- Real-time spreadsheet fetching (data updates via GitHub Action, not at runtime)
- More than 2 vehicles in head-to-head comparison
- Price data or TCO calculations (not in Bjørn's test data)
- Mobile app
- YouTube video links (nice-to-have for later, not in initial scope)
